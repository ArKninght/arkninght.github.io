WSL编译环境配置

在安装完成WSL之后当然要使用Linux的编程环境，本文主要介绍一下博主配置环境的过程以及遇到的一些问题。

1、conda安装
启动WSL，在用户目录下创建Download，进入Download文件夹

mkdir Download
cd Download

在Download目录下下载conda的安装包

wget https://repo.anaconda.com/archive/Anaconda3-2024.02-1-Linux-x86_64.sh

输入指令安装，一路enter，然后输入yes，再按一下enter，最后再输入yes

sh Anaconda3-2024.02-1-Linux-x86_64.sh # 安装刚刚下载的文件

此时使用conda指令会显示conda: command not found，退出当前登录重启Power Shell并重启WSL，显示(base) 用户名@DESKTOP-UA3C67F:~$即代表conda安装成功。

conda在安装完成之后在每次启动wsl时会自动激活，输入conda config --set auto_activate_base false指令以关闭启动自动激活激活

利用conda创建新的环境，用于深度学习编程。

conda create -n cuda118 python==3.8 # 创建cuda118环境并配置python3.8
conda activate cuda118 # 激活刚刚创建的环境

2、CUDA安装
对Ubuntu进行换源(或者直接添加)，方便国内网络访问时仍然可以以较快的速度下载。打开配置文件

sudo vim /etc/apt/sources.list # sudo是管理员命令，即以管理员的身份运行该指令

i进入编辑模式，将下列源复制进刚刚打开的源列表(可以将其他源都注释掉，博主这里没有选择注释)，按esc退出编辑模式，输入:wq保存并退出。

deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ jammy main restricted universe multiverse
deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ jammy-updates main restricted universe multiverse
deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ jammy-backports main restricted universe multiverse
deb http://security.ubuntu.com/ubuntu/ jammy-security main restricted universe multiverse

之后对软件包进行更新和升级。

sudo apt update
sudo apt upgrade

安装gcc相关的软件包。

sudo apt install build-essential

访问链接CUDA Toolkit 11.8 Downloads | NVIDIA Developer下载cuda11.8，选项在网页中已经选好。


此处选择使用源进行本地安装，按顺序输入网页中的指令即可成功安装cuda11.8。
在安装完成之后打开环境变量，

sudo vim ~/.bashrc

按i进入编辑模式，添加cuda的环境，按esc并输入:wq保存编辑

export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:/usr/local/cuda-11.8/lib64
export PATH=$PATH:/usr/local/cuda-11.8/bin
export CUDA_HOME=$CUDA_HOME:/usr/local/cuda-11.8

输入source ~/.bashrc刷新环境变量，输入nvcc -V查看cuda是否已经安装成功。若成功安装并配置环境则会显示当前cuda版本。

3、cudnn安装
访问cuDNN Archive | NVIDIA Developer选择适合的cudnn版本。博主选择了8.9.7版本，将文件下载到本地中并移动到WSL2系统中。这里建议移动到/home/用户/Download/文件夹下(即在下载cuda时创建的文件夹)。

索引到压缩包对应的目录，按序输入如下指令，解压压缩包，并将其中的文件复制到对应的cuda文件夹中进行文件的替换。

sudo tar -xvf cudnn-linux-x86_64-8.9.7.29_cuda11-archive.tar.xz
cd cudnn-linux-x86_64-8.9.7.29_cuda11-archive/
sudo cp -r lib/* /usr/local/cuda-11.8/lib64/
sudo cp -r include/* /usr/local/cuda-11.8/include/
sudo chmod a+r /usr/local/cuda-11.8/lib64/libcudnn*
sudo chmod a+r /usr/local/cuda-11.8/include/cudnn*

输入如下指令验证是否安装成功

cat /usr/local/cuda-11.8/include/cudnn_version.h | grep CUDNN_MAJOR -A 2

显示如下输出则为安装成功。

4、Torch安装
激活之前创建的cuda118环境，在环境中安装Torch。
从官网找到下载链接以前的PyTorch 版本 |PyTorch的
博主这里选择的是torch2.2版本，输入官网中的安装指令即可开始安装。

conda install pytorch==2.2.2 torchvision==0.17.2 torchaudio==2.2.2 pytorch-cuda=11.8 -c pytorch -c nvidia

测试是否安装成功，若显示如下结果则代表torch成功安装。

使用conda命令下载torch时会发现下载速度较慢并且容易断开链接，一种方法是通过更换下载源来解决问题。但由于conda的国内镜像需要配置的网站比较多，博主又很懒，所以直接使用科学上网解决问题，但是WSL默认使用NAT模式，与Win的网络不互通，因此在科学上网方面也需要一定的配置。

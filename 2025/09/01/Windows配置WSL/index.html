Windows配置WSL

本文主要记录博主在个人电脑上配置WSL的主要过程以及遇到的一些问题和解决方法。

1、什么是WSL

WSL是适用于 Linux 的 Windows 子系统，是Windows的一项功能，可以在Windows计算机上运行Linux环境而不需要单独的虚拟机或者双重启动。

2、Windows打开Linux环境

打开Win的控制面板

打开程序，在程序与功能中有一个启用或关闭Windows功能的按键，打开它会显示如下所示的窗口


勾选

  • 适用于Linux的Windows子系统
  • 虚拟机平台

之后按照提示重启电脑即可。

3、安装WSL并将其移动到其他盘(博主为E盘)

1、设置WSL的默认版本

wsl --set-default-version 2

2、创建文件夹
在目标盘创建文件夹

3、下载并Linux系统
利用如下指令查看WSL可以安装的Linux发行版本

wsl --list --online

博主安装的是Ubuntu-20.04，安装时可能会有进度条卡住的情况，耐心等待。

wsl --install -d Ubuntu-20.04

在安装之后正常会出现用户注册界面，博主在此处遇到了报错

这是由于没有安装最新的内核，需要对WSL进行手动更新，下载最新的内核。
下载链接：https://wslstorestorage.blob.core.windows.net/wslblob/wsl_update_x64.msi
安装完成后再次打开即会出现用户注册界面，输入用户名密码注册即可。之后按下Ctrl+D退出登录.
4、导出Linux系统至指定文件夹

wsl --export Ubuntu-20.04 E:\WSL\Ubuntu-20.04\Ubuntu-20.04.tar

该指令会将Ubuntu-20.04打包至目标文件夹，使用ls指令可以查看是否成功导出。

wsl --unregister Ubuntu-20.04

注销原本安装在C盘的的Ubuntu-20.04。
5、将导出的系统安装到指定文件夹

wsl --import Ubuntu-20.04 E:\WSL\Ubuntu-20.04 E:\WSL\Ubuntu-20.04\Ubuntu-20.04.tar --version 2

该指令会将Ubuntu-20.0安装到E盘指定目录而不是默认的C盘。安装完成之后会将WSL2发行版的文件存储在一个**虚拟磁盘映像文件（ext4.vhdx）**中。

安装完成之后即可启动WSL

wsl -d Ubuntu-20.04

默认登陆为root用户登录

使用su - pby登录到先前创建的用户pby，同时赋予用户管理员权限usermod -aG sudo pby。
6、修改默认登录用户
以管理员身份运行Power Shell，并运行如下指令

ubuntu2004 config --default-user pby

输入wsl发现默认以用户pby登录

至此则在Windows系统上安装完成Linux子系统。
关于conda环境的配置则参照之后的博文。

4、WSL更新

在使用Docker容器时需要用户下载最新版本的WSL，使用命令进行WSL的更新

wsl --update

但是使用命令对WSL进行更新的时候不知什么原因进度始终为0，为了快速更新，可以去往github上的WSL版本发行网站 https://github.com/microsoft/WSL 下载最新的发行版本。打开后点击安装即可立即安装最新的WSL版本。
